# Pretty Countdown

> Beautiful countdown widgets for life's important moments.

Pretty Countdown is an Android-first, **widget-driven** app. The home-screen
widget *is* the product — the in-app screens exist mainly to create and
configure events, choose a widget style, and manage notifications. A user who
sets up an event once and never opens the app again is the success case.

This project is a v1 implementation built from the product spec. Everything
is **local-first**: there is no account, login, or cloud sync. All data lives
in an on-device Room database and DataStore preferences, and "AI" features are
template-based suggestion engines that run entirely on-device.

## Tech stack

- **Kotlin** + **Jetpack Compose** (Material 3) for the in-app UI
- **Jetpack Glance** (`glance-appwidget`, `glance-material3`) for home-screen
  widgets
- **Room** for local persistence (events, checklist items, collections)
- **DataStore Preferences** for app settings, gamification stats, and
  per-widget configuration
- **WorkManager** for periodic widget refresh, milestone notifications, and
  yearly recurring-event rollover
- **Navigation Compose** for in-app navigation (5-tab bottom navigation)
- **Coil** for async photo loading
- **kotlinx.serialization** for JSON backup/restore

Build configuration: AGP 8.2.0, Kotlin 1.9.10, KSP 1.9.10-1.0.13, Gradle 8.4,
`compileSdk`/`targetSdk` 34, `minSdk` 26, Java 17 toolchain.

## App structure

Five bottom-navigation tabs (`ui/navigation/AppNavHost.kt`):

1. **Home** — upcoming events as cards with live countdowns
2. **Widgets Gallery** — preview and pick from the 11 widget styles
3. **Notifications** — upcoming/recent milestone reminders
4. **Collections** — group related events (e.g. "Wedding Planner") and view
   their combined countdown
5. **Settings** — theme, defaults, badges, and JSON backup/restore

Plus full-screen flows for **Create/Edit Event**, **Event Detail**
(notes, checklist, recurring toggle, collection membership), and
**Collection Detail**.

## Widget styles

Eleven Glance widget styles are implemented in
`widget/WidgetStyleRenderers.kt`, selectable per-event via the widget
configuration screen (`widget/WidgetConfigActivity.kt`):

| Style | Look |
| --- | --- |
| Minimal | Big number, clean typography, nothing else |
| Elegant | Soft serif vibes with an icon and subtitle |
| Flip Clock | Retro split-flap digit blocks |
| Circular Progress | A ring of dots fills as the date nears |
| Glassmorphism | Frosted glass card over a color wash |
| Cute Kawaii | Pastel colors, friendly emoji, "sleeps left" |
| Retro Pixel | Monospace, bracketed, 8-bit energy |
| Neon | Glowing digits on a midnight background |
| Calendar | A torn calendar page for the big day |
| Photo | The event photo fills the background |
| Journey | A little traveler walks toward the date along an emoji track |

Two widget types are registered (`countdown_widget_info.xml`,
`dashboard_widget_info.xml`):

- **Countdown widget** (`CountdownGlanceWidget`) — single event, configurable
  size (Small 2x2 → Large 4x4), style, and countdown format. Resolves to the
  last-viewed event by default.
- **Dashboard widget** (`DashboardGlanceWidget`) — XL widget showing a list of
  upcoming events at a glance, no per-instance configuration needed.

## Countdown formats

`data/model/CountdownFormat.kt`: Days Only, Days + Hours, Days + Hours +
Minutes, Sleeps Left, Weeks Remaining, Percentage Complete. Math lives in
`util/CountdownMath.kt` and is shared between the in-app UI and widgets.

## Categories

`data/model/EventCategory.kt` ships 15 categories (Birthday, Wedding,
Anniversary, Travel, Vacation, Exam, Fitness Goal, Baby Arrival, Festival, New
Year, Product Launch, Goal, Habit & Streak, Personal Milestone, Custom), each
with a default emoji and color palette (`data/model/ColorPalette.kt`).

## Background work

`work/DailyMaintenanceWorker.kt` runs as a 15-minute `PeriodicWorkRequest`
(the WorkManager minimum) and handles:

- **Smart milestone notifications** — friendly copy generated by
  `util/SmartNotifications.kt` at meaningful countdown checkpoints, plus
  user-defined custom milestones
- **Recurring events** — yearly events automatically roll forward to next
  year's date once they pass (`EventRepository`)
- **Widget refresh** — keeps all home-screen widgets in sync
  (`widget/WidgetUpdater.kt`), also triggered immediately after manual saves,
  edits, and deletes

## Gamification

`util/Badges.kt` defines unlockable badges (first event, 5/10/100 events
created, 3-day/7-day open streaks) based on stats tracked in
`data/SettingsRepository.kt` and surfaced on the Settings screen.

## On-device "AI"

`util/AISuggestions.kt` provides curated, template-based suggestions (e.g.
event names, milestone ideas, motivational copy) driven entirely by local
logic — **no network calls, no LLM, no cloud API**. This keeps the app
fully functional offline and consistent with the local-first principle.

## Backup & restore

Settings → Backup lets users export all events, checklist items, and
collections to a single JSON file (`data/model/BackupData.kt`, via
`kotlinx.serialization`) using the Storage Access Framework, and restore from
a previously exported file. `xml/backup_rules.xml` and
`xml/data_extraction_rules.xml` also include the Room database and DataStore
files in Android's automatic backup/device-transfer flows.

## Roadmap (not in v1)

These spec items are intentionally deferred:

- **Weather integration** on widgets (e.g. showing forecast for outdoor
  events) — would require a network dependency and API key, which conflicts
  with the local-first v1 scope
- **Lock screen widgets** (Android 13+ `glance-appwidget` lock screen
  surfaces) — left as a follow-up once the home-screen widget set is
  validated

## Known limitations / risks

- **Not build-verified.** This project was developed in a sandbox without an
  Android SDK or Maven/Google Maven network access, so it has **not** been
  compiled, run, or tested. Open it in Android Studio and run a Gradle sync
  + build before relying on it.
- **Some Glance APIs are used based on documentation/training knowledge but
  not verified against the exact `glance-appwidget:1.0.0` /
  `glance-material3:1.0.0` artifacts**, including:
  - `androidx.glance.unit.ColorProvider`
  - `androidx.glance.appwidget.cornerRadius`
  - `androidx.glance.text.FontFamily.Monospace`
  - `androidx.glance.appwidget.lazy.LazyColumn` / `items`
  - `androidx.glance.layout.defaultWeight()`
  - `androidx.glance.appwidget.action.actionStartActivity`
  - `androidx.glance.action.clickable` / `actionParametersOf` and the
    `ActionParameters.Key<T>.name` property used to read Intent extras

  If a Gradle build reports unresolved references in
  `widget/WidgetStyleRenderers.kt`, `widget/CountdownGlanceWidget.kt`,
  `widget/DashboardGlanceWidget.kt`, or `widget/WidgetContent.kt`, these are
  the most likely spots to check against the installed Glance version.
